# BMM 문서

BMad Method Module (BMM) 완벽 가이드 - 프로젝트 복잡도에 맞춰 적응하는 AI 기반 애자일 개발 워크플로우.

---

## 🚀 시작하기

**BMM이 처음이신가요?** 여기서 시작하세요:

- **[퀵 스타트 가이드(Quick Start Guide)](./quick-start.md)** - 첫 프로젝트 구축을 위한 단계별 가이드 (15분 분량)
  - 설치 및 설정
  - 4단계(Four phases) 이해하기
  - 첫 워크플로우 실행
  - 에이전트 기반 개발 흐름

**빠른 경로:** 설치(Install) → workflow-init → 에이전트 안내 따르기

### 📊 시각적 개요

**[전체 워크플로우 다이어그램](./images/workflow-method-greenfield.svg)** - BMad 방법론 표준 그린필드(greenfield) 트랙의 모든 단계, 에이전트(색상 구분), 의사결정 지점을 보여주는 시각적 흐름도입니다.

## 📖 핵심 개념

BMM이 사용자의 필요에 어떻게 적응하는지 이해하기:

- **[규모 적응형 시스템(Scale Adaptive System)](./scale-adaptive-system.md)** - 프로젝트 크기와 복잡도에 따라 BMM이 적응하는 방식 (42분 분량)
  - 3가지 기획 트랙 (Quick Flow, BMad Method, Enterprise Method)
  - 트랙 자동 추천
  - 트랙별 문서 요구사항
  - 기획 워크플로우 라우팅

- **[BMAD 퀵 플로우(BMAD Quick Flow)](./bmad-quick-flow.md)** - 패스트트랙 개발 워크플로우 (32분 분량)
  - 3단계 프로세스: 사양(spec) → 개발(dev) → 선택적 검토(optional review)
  - 버그 수정 및 작은 기능 구현에 최적
  - 프로덕션 품질의 신속한 프로토타이핑
  - 며칠이 아닌, 몇 시간 만에 구현 완료
  - Barry (Quick Flow Solo Dev) 에이전트 전담

- **[퀵 플로우 솔로 개발자(Quick Flow Solo Dev) 에이전트](./quick-flow-solo-dev.md)** - 신속한 개발을 위한 엘리트 솔로 개발자 (18분 분량)
  - Barry는 자율적인 실행에 특화된 엘리트 개발자입니다.
  - BMAD Quick Flow 워크플로우를 완벽하게 체화하고 있습니다.
  - 구상부터 배포까지 무자비한 효율성으로 프로젝트를 진행합니다.
  - 인계(handoff)나 지연 없이, 순수하게 개발에만 집중합니다.

## 🤖 에이전트 및 협업

BMM의 AI 에이전트 팀에 대한 완벽 가이드:

- **[에이전트 가이드(Agents Guide)](./agents-guide.md)** - 포괄적인 에이전트 참조 문서 (45분 분량)
  - 12명의 전문 BMM 에이전트 + BMad Master
  - 에이전트 역할, 워크플로우 및 사용 시기
  - 에이전트 커스터마이징 시스템
  - 모범 사례 및 일반적인 패턴

- **[파티 모드 가이드(Party Mode Guide)](./party-mode.md)** - 다중 에이전트 협업 (20분 분량)
  - 파티 모드 작동 방식 (19개 이상의 에이전트가 실시간으로 협업)
  - 사용 시기 (전략적, 창의적, 다기능적, 복잡한 과제)
  - 파티 구성 예시
  - 다중 모듈 통합 (BMM + CIS + BMB + 커스텀)
  - 파티 모드에서의 에이전트 커스터마이징
  - 모범 사례

## 📋 워크플로우 가이드

단계별로 정리된 모든 BMM 워크플로우에 대한 포괄적 문서:

- **[Phase 1: 분석 워크플로우](./workflows-analysis.md)** - 선택적 탐색 및 연구 워크플로우 (595줄)
  - brainstorm-project, product-brief, research 등
  - 분석 워크플로우 사용 시기
  - 창의적이고 전략적인 도구들

- **[Phase 2: 기획 워크플로우](./workflows-planning.md)** - 규모 적응형 기획 (967줄)
  - prd, tech-spec, gdd, narrative, ux
  - 트랙 기반 기획 접근법 (Quick Flow, BMad Method, Enterprise Method)
  - 어떤 기획 워크플로우를 사용할 것인가

- **[Phase 3: 솔루셔닝 워크플로우](./workflows-solutioning.md)** - 아키텍처 및 검증 (638줄)
  - architecture, create-epics-and-stories, implementation-readiness
  - V6: 더 나은 품질을 위해 아키텍처 수립 **후**에 에픽(Epics) 생성
  - BMad Method 및 Enterprise Method 트랙에 필수
  - 에이전트 충돌 방지

- **[Phase 4: 구현 워크플로우](./workflows-implementation.md)** - 스프린트 기반 개발 (1,634줄)
  - sprint-planning, create-story, dev-story, code-review
  - 전체 스토리 수명주기
  - 한 번에 하나의 스토리 처리(One-story-at-a-time) 원칙

- **[테스트 및 QA 워크플로우](./test-architecture.md)** - 포괄적인 품질 보증 (1,420줄)
  - 테스트 전략, 자동화, 품질 게이트
  - TEA 에이전트 및 테스트 힐링(test healing)
  - BMad 통합 모드 vs 독립형 모드

**총계: 모든 단계에 걸쳐 34개의 워크플로우 문서화됨**

### 고급 워크플로우 참조

특정 복잡한 워크플로우에 대한 상세 기술 문서:

- **[프로젝트 문서화 워크플로우 참조](./workflow-document-project-reference.md)** - 기술 심층 분석 (445줄)

  - 스캔 레벨, 재개 가능성(resumability), 즉시 작성(write-as-you-go)
  - 다중 파트 프로젝트 감지
  - 정밀 분석을 위한 딥다이브 모드

- **[아키텍처 워크플로우 참조](./workflow-architecture-reference.md)** - 의사결정 아키텍처 가이드 (320줄)

  - 새로운 패턴 설계
  - 에이전트 일관성을 위한 구현 패턴
  - 적응형 촉진(facilitation) 접근법
